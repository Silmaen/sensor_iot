# BH1750 Ambient Light Sensor Module

## Overview

The BH1750 module provides **calibrated illuminance** (lux) via a Rohm BH1750FVI digital light sensor over I2C.
Unlike the analog photoresistor (`HAS_LIGHT`), this sensor outputs a calibrated 16-bit lux value directly — no ADC
pin required, no conflict with battery monitoring on A0.

Enabled by the `-DHAS_BH1750` build flag in `platformio.ini`.

**Metrics published** (on `thermo/{id}/sensors`):

| Metric | Unit | Range       | Resolution |
|--------|------|-------------|------------|
| `lux`  | lx   | 1-65535 lux | 1 lx       |

## Sensor Specifications

| Parameter         | Value                                       |
|-------------------|---------------------------------------------|
| Sensor IC         | Rohm BH1750FVI                              |
| Interface         | I2C                                         |
| I2C address       | 0x23 (ADDR LOW/float) or 0x5C (ADDR HIGH)  |
| Supply voltage    | 2.4-3.6V                                    |
| Current (active)  | ~120 µA (during measurement)                |
| Current (standby) | ~1 µA (power-down after one-shot read)      |
| Spectral response | ~560 nm peak (close to human eye response)  |
| Measurement time  | ~180 ms (high-resolution mode)              |
| Accuracy          | ±20% (typical), variation by light angle    |

## Wiring

### ESP8266 (Wemos D1 Mini)

| D1 Mini Pin | GPIO  | BH1750 Pin | Notes       |
|-------------|-------|------------|-------------|
| D1          | GPIO5 | SCL        | I2C clock   |
| D2          | GPIO4 | SDA        | I2C data    |
| 3V3         | —     | VCC        | 3.3V supply |
| GND         | —     | GND        | Ground      |
| GND         | —     | ADDR       | Address 0x23 (or leave floating) |

> **Note:** The BH1750 shares the I2C bus with BME280 (0x76), SHT30 (0x44), and other I2C sensors.
> All can coexist — no address conflicts.

### ESP32-C3

| ESP32-C3 Pin | GPIO  | BH1750 Pin | Notes       |
|--------------|-------|------------|-------------|
| GPIO8        | SDA   | SDA        | I2C data    |
| GPIO9        | SCL   | SCL        | I2C clock   |
| 3V3          | —     | VCC        | 3.3V supply |
| GND          | —     | ADDR       | Address 0x23 |

## I2C Address Configuration

| ADDR Pin | I2C Address | Notes             |
|----------|-------------|-------------------|
| GND/float | **0x23**    | Default           |
| VCC      | **0x5C**    | Alternate address |

Two BH1750 sensors can coexist on the same I2C bus using different addresses.

## Measurement Mode

The driver uses **one-time high-resolution mode** (opcode `0x20`):
- Resolution: 1 lux
- Measurement time: ~120-180 ms
- After measurement, the sensor **automatically powers down** (~1 µA)
- No continuous polling — ideal for battery-powered nodes

## Lux Reference Values

| Environment           | Typical lux |
|-----------------------|-------------|
| Moonlight             | 0.5-1       |
| Street lighting       | 10-20       |
| Living room           | 100-300     |
| Office, well-lit      | 300-500     |
| Overcast daylight     | 1000-2000   |
| Full daylight         | 10000-25000 |
| Direct sunlight       | 32000-65535 |

## PlatformIO Configuration

### Build Flag

```ini
build_flags = -DHAS_BH1750
```

> **Note:** `HAS_BH1750` and `HAS_LIGHT` are independent. `HAS_BH1750` uses I2C (no ADC),
> `HAS_LIGHT` uses an analog pin. Both can be enabled simultaneously if needed.

### Example Environment

```ini
[env:sensor_esp_sht30_lux]
extends = common_esp8266
build_flags =
    ${common_esp8266.build_flags}
    -DDEVICE_ID='"thermo_lux"'
    -DMQTT_DEVICE_TYPE='"thermo"'
    -DHAS_SHT30
    -DHAS_BH1750
    -DHAS_BATTERY
    -DHAS_DEEP_SLEEP
    -DHAS_CALIBRATION
```

### Module API

```c++
void bh1750_module_register(ModuleRegistry& reg);
void bh1750_module_contribute(PayloadBuilder& pb, float lux);
```

### Driver API

```c++
class Bh1750Sensor {
    bool begin();              // Init with default address 0x23
    bool begin(uint8_t addr);  // Init with specific address
    float read_lux();          // One-shot read, returns lux or -1.0f on failure
};
```

## BH1750 vs Analog Photoresistor

| Feature          | BH1750 (`HAS_BH1750`) | Photoresistor (`HAS_LIGHT`) |
|------------------|------------------------|-----------------------------|
| Output           | Calibrated lux (1-65535) | Relative % (0-100)         |
| Interface        | I2C (digital)          | Analog (ADC pin)            |
| ADC conflict     | **None**               | Shares A0 with battery      |
| Accuracy         | ±20%                   | Uncalibrated                |
| Power (standby)  | ~1 µA                  | 0 µA (passive)              |
| Price             | ~1.50 EUR              | ~0.50 EUR                   |

## Troubleshooting

| Symptom            | Cause                   | Fix                                                |
|--------------------|-------------------------|----------------------------------------------------|
| Init failed        | Wrong I2C address       | Check ADDR pin wiring (GND = 0x23, VCC = 0x5C)    |
| Always reads 0     | Sensor in power-down    | Driver handles power-on automatically — check I2C  |
| Reads max (65535)  | Direct sunlight / glare | Expected behavior at saturation                    |
| Unstable readings  | I2C noise / long wires  | Add 4.7k pull-ups on SDA/SCL, keep wires short    |
